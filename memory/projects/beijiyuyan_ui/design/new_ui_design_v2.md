# 北京语燕 - 新版UI设计文档

## 一、核心功能架构

### 1. 首页 (Home)
**顶部**：搜索栏 + 定位城市
**地图区域**：
- 显示当前城市地图
- 标记附近景点（不同颜色区分类型）
- 推荐路线高亮显示

**Tab导航**：
- 推荐路线（基于偏好智能推荐）
- 附近景点（距离排序）
- 摄影点（摄影师推荐）
- 小众秘境（平台独家）

**路线卡片**：
- 封面图 + 标题
- 标签（摄影/美食/亲子等）
- 时长 + 距离
- 热度指数（点击量/收藏量）
- 用户评分

---

### 2. 路线规划 (Plan) - 4步流程

#### Step 1: 选择时间
- 日期选择器
- 时长选项：周末2天 / 小长假3-5天 / 长假7天+
- 出行季节提示（最佳旅行时间）

#### Step 2: 重点目的地（支持多选）
- 搜索框输入目的地
- 热门目的地推荐
- 已选目的地列表（可拖拽排序）
- 智能提示："根据您的选择，建议游玩顺序为..."

#### Step 3: 旅行偏好（多维度）

**3.1 旅行节奏**（单选）
- 🌿 轻松休闲 - 每天1-2个景点，慢慢体验
- ⚡ 充实打卡 - 每天3-5个景点，行程较满
- 🎯 深度沉浸 - 少而精，体验当地生活

**3.2 旅行主题**（多选）
- 📷 摄影旅拍
- 🍜 美食探索
- 🏛️ 历史文化
- 🏔️ 自然风光
- 🏙️ 城市漫步
- 🥾 户外探险
- 👨‍👩‍👧‍👦 亲子家庭
- 💕 浪漫情侣

**3.3 旅行风格**（单选）
- 💎 小众独特 - 避开人潮，探索冷门
- 🔥 网红打卡 - 热门景点，社交分享
- ✨ 品质舒适 - 注重体验质量
- 💰 经济实惠 - 性价比优先

**3.4 旅行同伴**（单选）
- 🧍 独自旅行
- 💑 情侣二人
- 👨‍👩‍👧‍👦 家庭亲子
- 👫 朋友结伴

**3.5 特殊需求**（多选）
- ♿ 无障碍设施
- 🐕 宠物友好
- 🅿️ 停车方便
- 🍽️ 素食选择

#### Step 4: 生成路线
- AI智能规划
- 显示路线地图
- 每日行程详情
- 预计花费
- **生成笔记按钮** ⬅️ 新增功能

---

### 3. 足迹 (Footprint) - 全新改版

**顶部Tab切换**：

#### Tab 1: 我的足迹
- 地图模式：标记所有去过的地点
- 列表模式：时间线展示
- 统计：去过X个城市，Y个国家
- 成就徽章：旅行达人/摄影大师/美食家等

#### Tab 2: 旅行圈（新增）
**内容流**：
- 用户分享的优质照片和笔记
- 瀑布流布局（类似小红书）

**排序方式**：
- 🔥 热门（点击量+点赞量综合排序）
- ⭐ 精选（编辑精选）
- 🆕 最新（时间倒序）
- 📍 附近（距离排序）

**内容卡片**：
- 高质量图片（最多9张）
- 地点定位
- 简短描述
- 点赞/收藏/评论数
- 作者头像和昵称

**发布入口**：
- 发布照片/笔记
- 关联路线
- 添加标签

---

### 4. 生成笔记 (Note) - 新增功能

**触发方式**：路线详情页点击"生成笔记"

**笔记内容**：

#### 4.1 海报
- 自动选取路线中最美的3-5张照片
- 智能排版（多种模板可选）
- 添加文字：地点 + 日期 + 标语
- 品牌水印：北京语燕

#### 4.2 地图定位
- 显示完整路线地图
- 标记所有途经点
- **一键导航**：
  - 高德地图
  - 百度地图
  - Apple Maps
  - Google Maps

#### 4.3 重要提示
- 天气提醒
- 门票预约提示
- 最佳拍摄时间
- 当地注意事项
- 紧急联系信息

#### 4.4 分享功能
- 保存到相册
- 分享到微信/微博/小红书
- 生成链接分享

---

### 5. 个人中心 (Profile)

**用户信息**：
- 头像/昵称/等级
- 旅行统计：天数/城市/里程

**功能菜单**：
- 我的路线（收藏/创建）

## 四、数据来源与核实流程

### 景点数据
1. **官方来源**：景区官网、政府旅游网站
2. **地图数据**：高德/百度API
3. **用户UGC**：小红书、马蜂窝、穷游
4. **摄影作品**：500px、Instagram、图虫

### 核实流程
1. **定位核实**：地图坐标交叉验证
2. **照片核实**：反向图片搜索验证
3. **信息更新**：定期巡检更新状态
4. **用户反馈**：建立纠错机制

### 热门度算法
```
热度分 = 点击量×0.3 + 收藏量×0.25 + 点赞量×0.2 + 评论量×0.15 + 分享量×0.1
```

## 五、技术实现建议

### 前端
- 框架：React Native / Flutter（跨平台）
- 地图：高德地图SDK / Mapbox
- 图片处理：Canvas API

### 后端
- 路线规划算法：A*算法 + 用户偏好权重
- 推荐系统：协同过滤 + 内容推荐
- 图片存储：阿里云OSS / AWS S3

### 第三方服务
- 地图导航：高德/百度/腾讯地图API
- 天气数据：和风天气
- 支付：微信支付/支付宝
