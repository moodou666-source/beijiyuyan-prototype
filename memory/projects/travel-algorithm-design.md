# 旅游推荐算法设计文档

## 核心目标
构建真实、可信的旅游推荐系统，减少营销号影响，提供真实用户认可的打卡点和推荐。

## 数据来源

### 1. 微博数据
- **采集方式**: 微博API、爬虫
- **数据维度**:
  - 真实用户打卡照片（带地理位置）
  - 用户评论和点赞数
  - 发布时间和频率
  - 用户粉丝数（判断是否为营销号）
- **反营销号策略**:
  - 过滤粉丝数>10万且内容高度相似的账号
  - 优先采集普通用户（粉丝<1万）的内容
  - 检测重复内容和模板化文案

### 2. 抖音数据
- **采集方式**: 抖音API、爬虫
- **数据维度**:
  - 视频播放量、点赞数、评论数
  - 视频标签和地理位置
  - 用户互动率（评论/播放比）
- **反营销号策略**:
  - 过滤点赞数异常高但评论率低的视频
  - 优先采集真实用户拍摄的UGC内容
  - 检测商业推广标签

### 3. 小红书数据
- **采集方式**: 小红书API、爬虫
- **数据维度**:
  - 笔记点赞、收藏、评论数
  - 图片质量和真实性
  - 用户等级和活跃度
- **反营销号策略**:
  - 过滤"探店""合作"等商业标签
  - 优先采集"真实体验""避雷"等真实内容
  - 检测过度修图和滤镜

## 算法设计

### 真实性评分算法
```
真实性得分 = 基础分 × 用户可信度 × 内容原创度 × 互动质量

其中:
- 基础分: 内容本身的质量（图片清晰度、文字详细度）
- 用户可信度: 普通用户(1.0) > 小V(0.8) > 大V(0.5) > 营销号(0.1)
- 内容原创度: 检测重复内容，原创(1.0) > 改编(0.7) > 复制(0.2)
- 互动质量: 真实评论(1.0) > 模板评论(0.5) > 刷量评论(0.1)
```

### 推荐排序算法
```
推荐得分 = 真实性得分 × 热度系数 × 时效性系数 × 地理位置匹配度

其中:
- 热度系数: 近期互动增长趋势
- 时效性系数: 新内容(1.0) > 半年内(0.8) > 一年内(0.6) > 旧内容(0.3)
- 地理位置匹配度: 根据用户当前位置计算
```

### 反作弊机制
1. **账号黑名单**: 识别并标记营销号、水军账号
2. **内容去重**: 检测相似度>80%的内容，只保留最早发布的
3. **异常检测**: 识别短时间内大量点赞、评论的异常行为
4. **用户举报**: 建立用户反馈机制，人工审核可疑内容
